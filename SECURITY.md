# Security Policy

## Supported Project Scope

This repository is part of the ZeaZDev platform infrastructure portfolio. Security work should focus on infrastructure-as-code, deployment workflows, access patterns, and operational documentation.

## Reporting

Open a private security advisory or contact the project owner directly for sensitive reports. Do not disclose sensitive findings in public issues.

## Handling Guidelines

- Keep production-impacting changes reviewable.
- Avoid including private environment details in commits, issues, or pull requests.
- Use least-privilege access patterns.
- Prefer reproducible configuration and documented operational steps.

## Response Process

1. Triage the report.
2. Confirm affected components.
3. Prepare a minimal fix.
4. Validate the change.
5. Document the remediation.
