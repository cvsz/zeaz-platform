# User Feedback Playbook (Thai-first)

## Goal

Collect actionable onboarding feedback from first users without invasive tracking.

## Interview triggers

- User stops before first product save
- User does not generate AI content after adding product
- User does not return within 7 days
- User reports content quality or compliance confusion

## Thai interview script (10-15 minutes)

1. "เป้าหมายหลักที่อยากทำให้สำเร็จในวันนี้คืออะไร?"
2. "ขั้นตอนไหนที่รู้สึกช้าหรือสับสนที่สุด?"
3. "ตอนสร้างโพสต์ AI มีส่วนไหนต้องแก้เยอะเป็นพิเศษไหม?"
4. "ข้อความเปิดเผยว่าเป็นลิงก์ Affiliate ชัดเจนพอไหม?"
5. "ถ้าเลือกได้ 1 อย่างที่อยากให้ปรับทันที จะเป็นเรื่องไหน?"

## Activation friction form guidance (Thai)

- ปัญหาที่เจออยู่ขั้นตอนไหน (สมัคร / เพิ่มสินค้า / ลิงก์ Affiliate / สร้างโพสต์ / Export / HyperFrames)
- เจอผลกระทบอะไร (ทำต่อไม่ได้ / ใช้เวลานาน / ไม่มั่นใจความถูกต้อง)
- สิ่งที่คาดหวังให้ระบบช่วยคืออะไร

## Bug report template

- วันที่และเวลา
- หน้าที่พบปัญหา
- ขั้นตอนก่อนเกิดปัญหา
- ผลที่คาดหวัง
- ผลที่เกิดขึ้นจริง
- ภาพหน้าจอ (ถ้ามี)
- ระดับความเร่งด่วน (ต่ำ/กลาง/สูง)

## Feature request template

- เป้าหมายที่ต้องการให้ฟีเจอร์ช่วย
- ขั้นตอนปัจจุบันที่ทำอยู่
- ปัญหาที่เจอจากขั้นตอนปัจจุบัน
- แนวทางที่อยากได้จากระบบ
- ผลลัพธ์ที่คาดว่าจะดีขึ้น

## Feedback triage rules

- P0: security/compliance risk
- P1: blocks activation milestones #2-#5
- P2: quality-of-life improvements
- P3: nice-to-have experiments

## Reporting cadence

- Daily: top 3 friction points
- Weekly: activation + retention trend summary
- Bi-weekly: prioritized growth experiment backlog refresh
