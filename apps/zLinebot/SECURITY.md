# Security Policy (EN)

## Supported Versions
Security fixes are prioritized for the latest active branch.

## Reporting a Vulnerability
Please report vulnerabilities privately to maintainers (security email/DM channel used by this repository).

Include:
- Affected component/path
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

## Response Targets
- Acknowledgement: within 72 hours
- Initial triage: within 7 days
- Fix timeline: based on severity and exploitability

## Security Best Practices for Contributors
- Never commit secrets/API keys
- Preserve tenant isolation boundaries
- Validate input and auth checks
- Avoid logging sensitive personal/payment data
