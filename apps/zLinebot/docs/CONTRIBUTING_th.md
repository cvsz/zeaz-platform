# แนวทางการมีส่วนร่วม (TH)

## การตั้งค่าสำหรับพัฒนา
1. Fork และ clone โครงการ
2. สร้าง feature branch
3. คัดลอก env และรันระบบ:
   ```bash
   cp .env.example .env
   docker compose up -d --build
   ```
4. ตรวจสอบระบบด้วย `curl http://localhost:3000/health`

## ขอบเขตการมีส่วนร่วม
- แก้บั๊ก
- ปรับปรุงความปลอดภัย
- เพิ่มประสิทธิภาพ
- ปรับปรุงเอกสารและตัวอย่าง

## เช็กลิสต์ก่อนส่ง PR
- PR ควรเล็กและโฟกัสชัดเจน
- อธิบายเหตุผลและวิธีแก้
- แนบหลักฐานการทดสอบ (คำสั่งที่ใช้)
- อัปเดตเอกสารเมื่อพฤติกรรมระบบเปลี่ยน
- ตรวจผลกระทบต่อ tenant และ privacy

## รูปแบบ commit
ใช้ข้อความ commit ที่สื่อความหมาย เช่น:
- `feat: ...`
- `fix: ...`
- `docs: ...`

## แนวทางรีวิว
ผู้ดูแลจะพิจารณาความถูกต้อง ความปลอดภัย การดูแลรักษา และผลกระทบต่อระบบเดิม
