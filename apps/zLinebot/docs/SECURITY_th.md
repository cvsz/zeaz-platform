# นโยบายความปลอดภัย (TH)

## เวอร์ชันที่รองรับ
การแก้ไขช่องโหว่จะให้ความสำคัญกับ branch ล่าสุดที่ใช้งานอยู่

## การรายงานช่องโหว่
กรุณารายงานช่องโหว่ผ่านช่องทางส่วนตัวของผู้ดูแลโครงการ (security email/DM)

ข้อมูลที่ควรแนบ:
- ส่วนที่ได้รับผลกระทบ
- ขั้นตอนการทำซ้ำ
- ระดับผลกระทบ
- แนวทางบรรเทาเบื้องต้น (ถ้ามี)

## เป้าหมายการตอบสนอง
- ตอบรับรายงานภายใน 72 ชั่วโมง
- ประเมินเบื้องต้นภายใน 7 วัน
- กำหนดเวลาแก้ไขตามระดับความรุนแรง

## แนวปฏิบัติด้านความปลอดภัยสำหรับผู้ร่วมพัฒนา
- ห้าม commit secrets/API keys
- รักษา tenant isolation
- ตรวจสอบ input และ auth ให้ครบ
- หลีกเลี่ยงการ log ข้อมูลอ่อนไหวส่วนบุคคล/การชำระเงิน
