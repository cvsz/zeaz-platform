# VDev Initial Release Notes

## Release scope

This release creates the initial VDev static product surface in `apps/zdev`.

## Product coverage

| Area | Implementation |
| --- | --- |
| Welcome message | Hero section introduces VDev as an all-in-one development environment. |
| Code-server foundation | Product copy identifies code-server as the future workspace foundation without embedding credentials or automation. |
| Key features | Cards cover unified workspace, browser access, customizable tooling, and collaboration. |
| How it works | Timeline covers access, stack selection, coding, and team sharing. |
| Get started | Local-only readiness checklist supports launch planning. |
| Why choose VDev | Trust cards cover simplicity, flexibility, collaboration, accessibility, and security. |

## Known limitations

- The app is a static product surface and does not launch a live IDE.
- No production hostname, tunnel, or Cloudflare Access policy is configured in this change.
- Production readiness depends on a future reviewed deployment plan with identity, access, audit, and runtime controls.
