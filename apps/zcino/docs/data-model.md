# Data Model

## Persistence overview

The backend uses PostgreSQL as the system of record for catalog and tracking data. Redis is used as a cache for game catalog reads, and NATS is an optional event bus for click-event fan-out after successful tracking persistence.

## PostgreSQL objects

### `providers`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | UUID | Primary key, generated with `gen_random_uuid()`. |
| `name` | text | Unique, non-blank provider name. |
| `is_active` | boolean | Active provider flag; defaults to `true`. |
| `created_at` | timestamptz | Insert timestamp. |
| `updated_at` | timestamptz | Maintained by trigger where migration 001 is applied. |

Indexes:

- `idx_providers_active_name` on `(is_active, name)`.

### `games`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | UUID | Primary key. |
| `provider_id` | UUID | Required foreign key to `providers(id)`. |
| `name` | text | Non-blank game name. |
| `provider` | text | Denormalized provider name retained for API compatibility and filtering. |
| `category` | text | Non-blank category. |
| `rtp` | numeric(5,2) | Return-to-player value constrained between 0 and 100. |
| `volatility` | `game_volatility` enum | Allowed values are `low`, `medium`, and `high`. |
| `thumbnail_url` | text | Must match an HTTP(S) URL pattern. |
| `is_active` | boolean | Active game flag; read endpoints should return active games. |
| `created_at` | timestamptz | Insert timestamp. |
| `updated_at` | timestamptz | Maintained by trigger where migration 001 is applied. |

Indexes:

- `idx_games_active_provider_name` on `(is_active, provider, name)`.
- `idx_games_provider_id` on `(provider_id)`.
- `idx_games_active_category` on `(is_active, category)`.
- `idx_games_active_rtp` on `(is_active, rtp)`.

### `tracking_events`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | UUID | Primary key generated by the service before queueing. |
| `tenant_id` | text | Required, max 100 characters, defaults to `public`. |
| `event_type` | text | `impression` or `click`. |
| `game_id` | UUID | Foreign key to `games(id)`. |
| `session_id` | text | Required, max 128 characters. |
| `user_id` | text | Optional, max 128 characters. |
| `provider` | text | Optional denormalized provider. |
| `country` | char/text depending on migration history | Optional ISO alpha-2 code. New code uppercases incoming country values. |
| `placement` | text | Optional UI placement, max 100 characters. |
| `click_target` | text | Optional click target; constrained to click events only. |
| `affiliate_id` | text | Optional affiliate/publisher identifier. |
| `campaign_id` | text | Optional campaign identifier. |
| `referrer_url` | text | Optional referrer URL for attribution debugging. |
| `session_duration_ms` | bigint | Optional non-negative duration. |
| `occurred_at` | timestamptz | Event occurrence time, defaults in service when omitted. |
| `received_at` | timestamptz | Server receive time. |
| `metadata` | jsonb | Optional metadata map, defaults to `{}`. |
| `created_at` | timestamptz | Insert timestamp. |

Indexes:

- `idx_tracking_events_tenant_occurred_at` on `(tenant_id, occurred_at DESC)`.
- `idx_tracking_events_type_occurred_at` on `(event_type, occurred_at DESC)`.
- `idx_tracking_events_game_occurred_at` on `(game_id, occurred_at DESC)`.
- `idx_tracking_events_session_occurred_at` on `(session_id, occurred_at DESC)`.
- `idx_tracking_events_affiliate_occurred_at` on `(affiliate_id, occurred_at DESC)` where `affiliate_id IS NOT NULL`.
- `idx_tracking_events_campaign_occurred_at` on `(campaign_id, occurred_at DESC)` where `campaign_id IS NOT NULL`.
- `idx_tracking_events_country_occurred_at` on `(country, occurred_at DESC)` where `country IS NOT NULL`.

### `affiliate_clicks` view

`affiliate_clicks` exposes click-event attribution fields for affiliate and campaign reporting without changing the ingestion API. It filters `tracking_events` to rows where `event_type = 'click'` and projects event, game, user, affiliate, campaign, country, referrer, and timestamp fields.

## Cache ownership

Redis caches game catalog data behind the cache package. Cache keys and TTLs should be treated as implementation details owned by the catalog service. The default cache TTL is `5m` and should be tuned based on catalog freshness requirements.

## Event ownership

Click publication happens after tracking batches are successfully inserted. The published click event includes tenant, game, user, country, session, Unix time, and timestamp fields. Consumers should assume at-least-once semantics if a durable event publisher is introduced and should deduplicate on the tracking event identifier when that field is added to the event contract.

## Migration policy

- Apply migrations in numeric order.
- Keep migrations forward-only unless an explicit rollback strategy is documented in an operations change plan.
- Add indexes concurrently in large production databases when supported by the deployment process.
- Any API-visible schema change must update `docs/api.md`, this data model, frontend mapping code when applicable, and analytics consumer documentation.
- Backfills must be idempotent and observable. Include row counts, runtime expectations, and abort criteria in release notes.

## Data retention and privacy notes

- Tracking metadata is intentionally constrained to string keys and values. Do not store payment data, wallet data, regulated betting decisions, secrets, or raw identity documents in metadata.
- `referrer_url` can contain sensitive query strings. Production deployments should consider URL normalization or query redaction before storage.
- Tenant-specific retention, deletion, and export workflows are not fully implemented in the current repository and should be added before strict enterprise multi-tenant commitments.
