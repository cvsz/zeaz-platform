# Risk Halt Runbook

## Objective
Procedures for triggering and recovering from a global risk halt.

## Triggering a Halt
1. Ensure you have `MANAGE_RISK` permissions.
2. Execute the halt API endpoint or use the dashboard emergency button.
3. Verify that all automated trading stops immediately.

## Recovery
1. Investigate the cause of the risk anomaly.
2. Resolve the underlying issue.
3. Reset the halt flag via the API/Dashboard.
4. Verify systems resume gracefully.
