# ZAI Coder Integration

This document describes how `zai-coder` is integrated into the `zeaz-platform` monorepo as a git submodule.

## Structure

```
zeaz-platform/
├── apps/
│   └── zai-coder/          ← git submodule (github.com/cvsz/zai-coder)
├── infra/
│   └── zai-coder/
│       └── compose.yaml    ← Docker Compose service
└── Makefile                ← zai-coder-* targets
```

## Submodule Management

### Initial setup (after cloning zeaz-platform)
```bash
git submodule update --init --recursive
# or just:
make zai-coder-init
```

### Update zai-coder to latest main
```bash
make zai-coder-update
```

### Pin to a specific commit
```bash
cd apps/zai-coder
git checkout <commit-sha>
cd ../..
git add apps/zai-coder
git commit -m "chore(submodule): pin zai-coder to <commit-sha>"
```

## Development

### Install Python deps (into platform venv)
```bash
make zai-coder-install
```

### Run CLI
```bash
make zai-coder-run
# or directly:
cd apps/zai-coder && python3 -m zai_coder.cli --help
```

### Run tests
```bash
make zai-coder-test
```

## Docker

### Build image
```bash
make zai-coder-build
```

### Start service
```bash
make zai-coder-up
```

### View logs
```bash
make zai-coder-logs
```

## Environment Variables

Add these to the platform `.env` file as needed:

```env
# ZAI Coder
ZAI_CODER_PORT=8090
ZAI_LOG_LEVEL=INFO
```

AI provider keys (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`) are
passed through from the platform `.env` automatically.

## Connectivity

| App          | How it connects                        |
|--------------|----------------------------------------|
| `zdash`      | Via Traefik proxy at `zai.zeaz.dev`    |
| `zeaz-api`   | Direct HTTP at `http://zai-coder:8090` |
| `zchat`      | Via platform API gateway               |

## Production URL

`https://zai.zeaz.dev` → routed via Cloudflare Tunnel → Traefik → zai-coder container
